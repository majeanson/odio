AuthShell — Authenticated Layout Wrapper

active
odio-2026-047
The problem it solves

All authenticated pages need shared layout elements: the BottomTabBar, the UploadBanner (persistent upload progress), and eventually the OfflineBanner (global offline indicator). These cannot live at the page level — they'd need to be duplicated on every route. A single client-side shell mounts them once.

How it was solved

components/layout/AuthShell.tsx: 'use client'. Uses usePathname() to extract bandId/sessionId from URL segments. Renders: <UploadBanner />, <OfflineBanner /> (when ready), <div className='flex-1'>{children}</div>, <BottomTabBar bandId={...} sessionId={...} />.
❝ AuthShell is a client component even though most of its content is static
— usePathname() requires a client component. Without usePathname(), BottomTabBar cannot determine which tabs to show or hide, and routing context cannot be passed down. The cost of a client boundary here is minimal — AuthShell itself renders no heavy content.
❞
Caveats
  • bandId/sessionId extraction is regex-based on pathname — URL structure changes require updating the extraction logic

active

The problem it solves

How it was solved
